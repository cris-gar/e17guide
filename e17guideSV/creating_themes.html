Bodhi-Guiden till Enlightenment
Skapa Teman

av Agustin J. Verdegal T (Agust)

Denna artikel, E17: Create Your Own Custom Themes, publicerades ursprungligen i PCLinuxOs Magazine, Volym 51, April 2011 (Dessa länkar kräver anslutning till internet).

Stort tack till Agust för hans portning av sitt PcLinuxOs tema A-LaNoche till Bodhi och för att låta oss lägga upp en modifierad form av sin artikel här. Innan du börjar, Om det inte redan är installerat, behöver du installera A-LaNoche temat: 

sudo apt-get install bodhi-theme-lanoche


För kännedom, Bodhi installerar E17 teman i mappen /usr/share/enlightenment/data/themes/ men teman kan även installeras lokalt för bruk endast av nuvarande användaren i ~/.e/e/themes. För en introduktion till användandet av teman, se Teman avsnittet.

Inledning


Jag skall försöka förklara ett enkelt sätt att skapa ett E17 tema på genom att använda temat A-LaNoche som jag skapat. Det första som skall göras är att ta en kopia på temat och sen öppna kopian så vi kan börja jobba.

För att öppna ett tema använder vi oss av följande kommando. 

edje_decc (filnamn.edj)

Själv delar jag upp tema-arbetet i tre moment: bakgrundsbilden i .edj formatet, bilder i .png formatet, och .edc filen.

Om vi vill använda en enkel bakgrundsbild istället för en bakgrundsbild i .edj formatet, måste vi ändra .edc filen på detta vis. 


/*** DEFAULT WALLPAPER ***/
/** wallpaper original realizado por Agust gracias **/
images {
	image: "grill_dark_tiny_pattern.png" COMP;
}
group {
	name: "e/desktop/background";
	max: 1680 1050;
	parts {
		part {
			name: "background_image";
			description {
				state: "default" 0.0;
				image {
					normal: "grill_dark_tiny_pattern.png";
				}
			}
		}
	}
}


I detta exempel är den skalad till upplösningen 1680 x 1050. Döp om den till grill_dark_tiny_pattern.png och lägg den i tema mappen vi skapade.

På detta sätt får vi bakgrundsbilden till vårat tema.


Del 2. Bilder och Deras Innebörd


Ovan ser vi "Om Enlightenment" sidan som nås via Menu -> Enlightenment -> About. Den är uppbyggd av flera mindre bilder i .png format. I detta exempel ska jag försöka förklara de viktigaste.


Om du ser till bilden nedanför finner du about_bot.png, about_mid.png och about_top.png. De är bilderna som skapar bakgrunden till bilden ovan och utgör nedre delen, mitten och övre delen. Stäng knappen och krysset har även de .png filer.

De andra filerna du hittar här ser man oftast tydligt deras funktion, pil - ner, vänster, höger och upp. Filerna som börjar på "bat", alldeles riktigt, de har med datorns batteristatus att göra.


Nu kommer vi till en av de viktigaste bitarna i detta exempel, image base_bg.png. Detta är filen vi kommer att använda för huvudmenyn. Som du ser nedan är denna bild mycket mörk. För att texten skall få kontrast mot bakgrunden och synas, ändrar vi den till genomskinlig eller någon ljus färg. För att uppnå detta ändrar vi så här i default.edc filen. 


color_class {
name: "menu_item";
color: 221 221 221 255;
color3: 0 0 0 64;
}
color_class {
name: "menu_item_active";
color: 255 255 255 255;
color3: 0 0 0 14;


Som du ser har jag använt vitt, men du kan använda vilken färg du vill.


bd_bottom.png är bilden som används för nedre fönsterramen och bd_top.png för övre fönsterramen. bd_title_bg.png används av tittellisten när den är aktiv. Här har jag skapat en lätt genomskinlig glaseffekt.

bd_bottom bilderna är de som används för att minimera eller expandera fönstren.


Här ser vi ikonerna som används för bluetooth. Vi kan använda oss av vad vi vill bara vi håller oss till rätt bildstorlek så att vi slipper tvingas modifiera "default.edc" filen.
bnw.png är bilden som visar information om temat, vem som bidragit, osv.


Knappar använder sig av bilderna bt-base.png, base2.png och dis_base.png.
bt_sm_base1.png är bilden som används när vi skall klicka på en knapp. bt_sm_base2.png används när man väljer en draglist.

Här har du vårat exempel.

Del 3. Temats Beståndsdelar


I följande bild ser vi utöver .edc filen, en av de viktigaste delarna i skapandet av ett tema. Detta är build.sh skriptet. Det här: 


 #!/bin/sh edje_cc $@ -id . -fd . default.edc -o A-Noche.edj


är skriptet som skapade temat vi tittar på. Utöver själva byggandet, består temat av 10 bilder och fungerande fönster.

En av de delar jag särskillt gillar är klockan, vi använder den som exempel.

Jag utformade sekundvisaren som enbart en grön prick utan egentliga "pekare" så som de minut och timvisarna är.

Vi ser att temat innehåller fler bildfiler, men den viktigaste är default.edc filen som innehåller alla delar av vårat tema och genom vilken vi kan modifiera nästan allt.

Ett exempel är kalender modulen som jag lagt till temat genom att ändra i default.edc filen. För att använda kalendern, du ser den som (E17_calendar.png) måste du ändra så här: 


/*** MOD: CALENDAR ***/
images {
	image: "E17_calendar.png" COMP;
}
fonts {
	font: "VeraMono.ttf" "VeraMono";

	font: "VeraBd.ttf" "VeraBold";
}
group {
	name: "modules/calendar/main";
	parts {
		part {
			name: "calendar";
			description {
				state: "default" 0.0;
				aspect: 1.0 1.0;
				aspect_preference: BOTH;
				rel1 {
					relative: 0.0 0.0;
				}
				rel2 {
					relative: 1.0 1.0;
				}
				image {
					normal: "E17_calendar.png";
				}
			}
		}
		part {
			name: "monthday";
			type: TEXT;
			mouse_events: 0;
			description {
				state: "default" 0.0;
				rel1 {
					relative: 0.1 0.2;
					offset: 0 0;
					to: "calendar";
				}
				rel2 {
					relative: 1.0 0.7;
					offset: -1 -1;
					to: "calendar";
				}
				color: 46 52 54 255;
				color_class: "module_label";
				text {
					text: "??";
					font: "VeraBold";
					size: 13;
					min: 1 1;
					fit: 1 1;
					text_class: "module_large";
				}
			}
		}
		part {
			name: "weekday";
			type: TEXT;
			mouse_events: 0;
			description {
				state: "default" 0.0;
				rel1 {
					relative: 0.4 0.7;
					offset: 0 0;
					to: "calendar";
				}
				rel2 {
					relative: 0.7 0.9;
					offset: -1 -1;
					to: "calendar";
				}
				color: 46 52 54 255;
				color_class: "module_label";
				text {
					text: "??";
					font: "VeraMono";
					size: 9;
					min: 1 1;
					fit: 1 1;
					text_class: "module_large";
				}
			}
		}
	}
}


Utöver att ändra i filen måste vi även lägga till bilden e17_calendar.png, teckensnitten VeraMono.ttf och VeraBd.ttf. Så, nu har vi vår kalender. Värt att notera är även bildfilen dia_grad.png, den utgör grunden till detta tema. Härnäst ser vi bildfilerna som används av temat under inloggning (login manager), exq-bglight.png, exq-dot-glow.png, exq-dot.png och exq-logo.png. Vi ser även grill_dark_tiny_pattern.png vilket är bilden som normalt används som bakgrund ifall man inte specificerat en särskilld bakgrundsbild (som visades i del 1).

Följande bilder är temats ikoner. Du kan alltid byta ut dem mot andra som du kanske gillar bättre, bara du använder samma storlek och namn.

Här der vi bilderna för loggan i temat och även färgen som används vid markering av innehåll i huvudmenyn.

Del 4. Fler Bilder


I denna bildsektion ser vi ikonen som används för volym kontrol. Vi ser även bilderna för "skrivbordsbytaren", de är genomskinliga med en mörk bård.


För övergångseffekterna ser vi här bilderna för muspekaren och muspekareffekterna. dessa effekter kan se mucket bra ut.

Powerikonen har med batteriet att göra, när den är synlig indikerar det att batteriet laddas och försvinner när batteriet är fulladdat.

Nu tittar vi på färgen av hyllan som ska användas i temat. Notera att förvalda färgen på hyllan är densamma som den vi använde i huvudmenyn. E17 ger oss möjligheten att ändra så att färgen blir mer transparent, (det är vad jag gjorde). I det här exemplet har jag satt svart som val för hyllans färg.

Tittar vi vidare ser vi bilderna för skjutreglage, slider.png och slider_clicked.png. Jag har lagt till en ljuseffekt som gör att skjutknappen lyses upp när reglaget används. Ikonerna tacho_bg och tacho_dial_xx hör till E17 modulen: Cpufreq.

Slutligen ser vi bilderna som används för temperaturikonen. I temat används vgrad bilderna för E17 modulen: everything. Notera att ttf fonterna som används är samma som vi använde oss av i kalendern.

Del 5. Kompilera Temat


När vårat tema är klart behöver vi bara kompilera det. Om du använder dig av detta tema som grund, är kompilering mycket lätt.

Dubbelklicka på build.sh.

Om allt har gått väl ska en .edj tema fil ha skapats. I detta fall kommer filen att heta A-Noche.edj. Om det blir fel och filen inte skapas, beror detta på att det finns ett fel någonstans i ingående delarna i temat.

Det finns en del teman som använder enlightenment_remote. Dessa teman måste kompileras lite anorlunda. Öppna ett terminalfönster och skriv in följande. 

./build.sh
./build.sh && enlightenment_remote -restart
